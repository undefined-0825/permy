## 1. プロジェクトの目的とゴール (MUST)
* **最上位ゴール**: 1週間でプロダクトをリリースし、収益を上げること（最短で課金・継続利用へ繋げる）。
* **中核課題**: 夜職ユーザーの返信コストと心理的負担（罪悪感）の解消。
* **コア価値**: 「3案提示」「ユーザー主語」「自動送信禁止」。
* **判断基準**: 迷った際は常に「リリース速度」「UX（使い心地）」「有料化導線」を最優先し、技術的な美しさよりも実戦での有用性を重視する。

## 3. デバイス別「対話モード」の自動切り替え (MUST)
AIは回答生成前に必ずユーザーのデバイス情報をスキャンし、モードを自動選択すること。
* **スマホモード**: 外出先を想定。「一問一答形式」に徹し、回答は「1. 2. 3. の選択形式」とする。解説は3行以内。
* **PCモード**: 実装作業を想定。網羅的記述を許容するが、一度に大量の質問は禁止。優先度が高い順に確認を取る。

## 4. 思考・回答プロトコル (AIの自律性と誠実性)
* **自己訂正プロトコル**: 回答前に `/mnt/data/` を再スキャンし、最新Versionとの乖離があれば自己修正する。
* **記憶依存の禁止**: 長期メモリや過去の断片的な記憶を根拠にせず、必ずSpecファイル（物理事実）を唯一のソースとする。
* **推論・創作の禁止**: 仕様の根拠がSpecにない場合、勝手に補完せずユーザーに事実確認を求めること。
* **ステート表示**: 回答末尾に `[Current State]`（Active Spec / Task / Next Step）を表示する。

## 5. 事故防止・環境防衛 (HARD CONSTRAINT)
* **破壊的操作の三段階検証**: `git reset`, `rm`, 全置換等の前には「1.事実取得」「2.退避指示（ブランチ作成等）」「3.単一コマンド提示」を徹底する。
* **機密管理**: `.env` 等のGit混入を厳禁し、追跡済みファイルの除外は `git rm --cached` を提案する。
* **シェル整合**: PowerShell 5系に固定。構文の混在を避ける。

## 99. リポジトリ機密（APIキー等）混入時の対処（MUST）
**Added (JST):** 2026-03-01 15:00:00 +0900

### 99.1 禁止
- `.env` / `*.key` / 資格情報を **Gitにコミット禁止**。
- GitHub の Push Protection / Secret Scanning により **pushが拒否される** ため、作業が止まる。fileciteturn2file4L7-L24

### 99.2 もし一度でもコミットした場合（漏洩扱い）
- **当該キーは無効化して再発行**（ローテーション）する。fileciteturn2file4L20-L24
- 履歴から除去する（`git rm --cached` / 履歴rewrite 等）。  
  ※ 初期段階で外部共有が無い場合は「履歴リセット（.git作り直し）」が最短。ただし破壊的操作なので、実行前に影響/手順/ロールバックを明記し承認を取る。fileciteturn2file4L7-L24

### 99.3 予防（必須チェック）
- `.gitignore` に `backend/.env` 等を必ず含める（リポジトリ直下で管理）。
- `git status` で `.env` が **Untracked** であることを確認してから push する。
