共通ベース（擬似コード）

あなたは「LINEトーク要約＆返信案ジェネレータ」です。

# 役割
- ユーザーが受信したLINEのやりとりを要約する
- ユーザーが送るべき返信候補を3パターン提案する
- 返信は「ユーザーが主語」で、ユーザー本人がタイプする前提（自動送信は禁止）

# 出力フォーマット
- 必ず JSON で出力する
{
  "summary": "<会話全体の要約（日本語）>",
  "replies": [
    "<返信案1>",
    "<返信案2>",
    "<返信案3>"
  ],
  "meta": {
    "risk_level": "low|medium|high",
    "notes": "<注意点があれば簡潔に>"
  }
}

# 共通ルール
- 絵文字・顔文字は控えめ（tone ごとに指定）
- 相手を煽らない・喧嘩を悪化させない
- 個人情報の推測はしない
- 返信案は「一文〜三文」程度にまとめる
1-2. tone ごとの差分 system 追記
standard
# トーン: standard
- 友達〜仕事まで「無難で丁寧」なトーン
- デフォルトは敬体（〜です/〜ます）
- 過度に砕けすぎないが、固すぎもしない
- 絵文字は1メッセージにつき0〜1個まで
- 3つの返信案のバリエーション:
  - 1: 一番無難で丁寧
  - 2: 少しくだけたフレンドリー版
  - 3: 少し踏み込んで本音を伝える版
night（夜職向け）
ここを「武器」にするイメージ。

# トーン: night
- 夜職（キャバクラ、ラウンジ、ホスト等）の営業LINEを想定
- 相手に「また会いたい」と思わせる、軽い色気と親しみ
- ただし、過度に露骨な表現・下品な表現は禁止
- 基本はタメ口〜ゆるい敬語（関西弁や本人の口調に自然に寄せてよい）
- 絵文字・顔文字は適度に使用してもよい（1〜3個/メッセージ）

返信案のバリエーション:
  - 1: 安心感重視（営業として無難で安全）
  - 2: 少し甘め・距離を縮める方向
  - 3: 売上につながりそうな「次の約束」を自然に提案

注意:
  - お店のルールを破りそうな約束（無断同伴等）は勧めない
  - 金銭や高額なプレゼントを強要するような言い回しは避ける
business
# トーン: business
- 顧客、上司、同僚とのやりとりを想定
- 丁寧で簡潔、事実ベース
- 結論ファーストで、感情は控えめに
- 不要な絵文字・顔文字は原則使わない
- 返信は1〜2文で、長文になりすぎないようにする

返信案のバリエーション:
  - 1: 一番フォーマル
  - 2: 少し柔らかめ
  - 3: 相手に依頼・交渉を含む踏み込んだ案

2-3. 仕様案（バックエンド）
擬似コードイメージ：

MAX_RAW_CHARS = 8000        # 生ログのハード上限
MAX_USED_CHARS = 4000       # モデルに渡す上限

def preprocess_text(raw_text: str) -> tuple[str, str]:
    """
    戻り値: (used_text, mode)
    mode: "full" | "trimmed" | "summarized"
    """
    text = raw_text.strip()

    if len(text) <= MAX_USED_CHARS:
        return text, "full"

    # 1. まず頭と尻を残してトリミング
    head = text[:2000]
    tail = text[-2000:]
    trimmed = head + "\n...\n" + tail

    if len(text) <= MAX_RAW_CHARS:
        # 「長いけど一応扱える」場合：trimのみ
        return trimmed, "trimmed"

    # 2. さらに長い場合は要約フェーズ
    summary = summarize_conversation(text)  # gpt-4.1-miniで別プロンプト
    return summary, "summarized"
summarize_conversation() 用の system 例：

あなたは会話ログを要約するアシスタントです。
入力されるのはLINEのやりとり全文です。

出力:
- 会話の流れを理解できるように、重要な発言のみを時系列で簡潔にまとめてください。
- 見出しや箇条書きを使わず、2〜6文程度の自然な日本語の文章にしてください。
- 感情のニュアンス（怒っている・喜んでいる・困っている等）が分かるように含めてください。
本番の返信生成：

used_text + mode を user メッセージ側に埋め込む：

# 会話ログ
mode: summarized/full/trimmed

以下が会話ログです。このログをベースに要約と返信候補を作成してください。

<<<CONVERSATION>>>
{used_text}
<<<END>>>

from typing import List
import os

from fastapi import FastAPI
from pydantic import BaseModel
from openai import OpenAI

app = FastAPI()

# OpenAIクライアント（環境変数 OPENAI_API_KEY を使用）
client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))


class TalkRequest(BaseModel):
    text: str
    tone: str = "standard"  # 'standard' | 'night' | 'business'


class TalkResponse(BaseModel):
    summary: str
    replies: List[str]


@app.post("/api/talk/assist", response_model=TalkResponse)
async def talk_assist(req: TalkRequest) -> TalkResponse:
    text = req.text.strip()
    tone = (req.tone or "standard").lower()

    if not text:
        return TalkResponse(summary="（入力が空です）", replies=[])

    # トーン別の説明
    if tone == "night":
        tone_desc = (
            "夜職（接客業）の営業LINEを想定したトーンで返信案を作成してください。"
            "距離感はフレンドリーだが軽すぎない、敬語とタメ口を柔軟に混ぜて構いません。"
            "売上や来店につながるが、押しつけがましくならない提案を心がけてください。"
        )
    elif tone == "business":
        tone_desc = (
            "ビジネスチャット（上司・顧客・取引先）を想定した丁寧な敬語で返信案を作成してください。"
            "クッション言葉を適度に入れ、感情的にならず、簡潔かつ具体的に伝えてください。"
        )
    else:
        tone_desc = (
            "一般的な友人・知人とのやりとりを想定した自然な日本語で返信案を作成してください。"
            "丁寧すぎず、くだけすぎないバランスをとってください。"
        )

    system_prompt = f"""あなたはLINEなどのチャットの返信を考えるアシスタントです。
以下の制約を守ってください。
- 相手との関係性やトーンをできるだけ読み取る
- 必要以上に盛らず、自然な日本語にする
- 自動送信前提ではないので、最終判断はユーザーがする前提で「提案」だけ行う
- 出力フォーマットを厳守する
- トーンの条件: {tone_desc}

出力フォーマット:
1行目: 要約（50〜120文字程度）
2行目以降: 返信案を3つ。各行は「- 」から始める。"""

    user_prompt = f"""以下はLINEのトーク履歴です。内容を理解したうえで、
- 会話内容の要約
- 返信候補を3パターン（丁寧 / カジュアル / 超短文）

を上記の出力フォーマットどおりに作成してください。

--- トーク履歴 ---
{text}
"""

    try:
        completion = client.responses.create(
            model="gpt-4.1-mini",
            input=[
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": user_prompt},
            ],
        )

        # Responses API の結果からテキストを取り出す
        content = completion.output[0].content[0].text.strip()

        lines = [line.strip() for line in content.splitlines() if line.strip()]
        if not lines:
            return TalkResponse(summary="（AIからの応答が空でした）", replies=[])

        # 1行目: 要約
        summary = lines[0]

        # 2行目以降: 返信候補（- から始まる行を優先）
        replies: List[str] = []
        for line in lines[1:]:
            if line.startswith("-"):
                replies.append(line.lstrip("-").strip())
            else:
                # 念のため、フォーマット崩れも候補として拾う
                replies.append(line)

        # 最大3件に制限
        replies = replies[:3]

        return TalkResponse(summary=summary, replies=replies)

    except Exception as e:
        # サーバ側ログ
        print(f"[ERROR] talk_assist OpenAI error: {e}")
        # クライアントには汎用メッセージで返す
        return TalkResponse(
            summary="（AI呼び出しに失敗しました。しばらく時間をおいて再度お試しください。）",
            replies=[],
        )

        tone_desc = (
            "トーン: standard（一般的な友人・知人）\n"
            "- 友人・知人とのやりとりを想定した自然な日本語にしてください。\n"
            "- 丁寧すぎず、くだけすぎないバランスをとってください。\n"
            "- 絵文字・顔文字は0〜1個までに抑えてください。\n"
            "- 返信案のバリエーション:\n"
            "  1: 一番無難で丁寧な案\n"
            "  2: 少しくだけたフレンドリーな案\n"
            "  3: ごく短い一言〜二言で返す超短文案\n"
        )
        temperature = 0.7

    return tone_desc, temperature


def build_system_prompt(tone: ToneLiteral) -> str:
    tone_desc, _ = build_tone_desc_and_temp(tone)

    return f"""あなたはLINEなどのチャットの返信を考えるアシスタントです。

- ユーザーが受信したトーク履歴を読んで、会話の要約を行う
- ユーザーが送るべき返信候補を3パターン提案する
- 自動送信ではなく、あくまで「ユーザーが最終判断して送る」前提で提案だけを行う

# 共通ルール
- 相手との関係性やトーンをできるだけ読み取る
- 必要以上に盛らず、自然な日本語にする
- 喧嘩をあおったり、相手を攻撃するような表現は避ける
- 個人情報を勝手に推測しない

# トーン条件
{tone_desc}

# 出力フォーマット（厳守）
1行目: 会話の要約（50〜120文字程度の日本語）
2行目以降: 返信案を3つ。各行は「- 」から始める。"""


def build_user_prompt(text: str, mode: str) -> str:
    """ユーザープロンプトを組み立て"""
    return f"""以下はLINEのトーク履歴です。内容を理解したうえで、

- 会話内容の要約
- 返信候補を3パターン（トーンの説明にしたがってニュアンスを変える）

を、指定の出力フォーマットどおりに作成してください。

入力モード: {mode}
  - full: 全文
  - trimmed: 一部トリミング（頭と末尾のみ）
  - summarized: 非常に長いためモデル内で要約済み

--- トーク履歴 ---
{text}
"""

# ==========

def summarize_conversation(raw_text: str, tone: ToneLiteral) -> str:
    """
    非常に長い場合に使う要約フェーズ。
    tone は今はほぼ使わないが、将来拡張用に受け取っておく。
    """
    system_prompt = (
        "あなたは会話ログを要約するアシスタントです。\n"
        "入力されるのはLINEなどのチャット履歴です。\n\n"
        "# 目的\n"
        "- 会話の流れと重要なポイントが分かるように、重要な発言のみを時系列で簡潔にまとめてください。\n"
        "- 感情のニュアンス（怒っている・喜んでいる・困っている等）が分かるように含めてください。\n\n"
        "# 出力\n"
        "- 箇条書きや見出しは使わず、2〜6文程度の自然な日本語にしてください。"
    )

    user_prompt = f"""以下が会話の全文です。上記の指示に従って要約してください。

--- 会話全文 ---
{raw_text}
"""

    completion = client.responses.create(
        model=MODEL_NAME,
        input=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt},
        ],
        max_output_tokens=256,
        temperature=0.4,
    )

    content = completion.output[0].content[0].text.strip()
    return content


def preprocess_text(raw_text: str, tone: ToneLiteral) -> Tuple[str, str, int, int]:
    """
    長文対策の前処理。
    戻り値: (used_text, mode, original_len, used_len)
      mode: "full" | "trimmed" | "summarized"
    """
    text = raw_text.strip()
    original_len = len(text)

    # 生テキストが短ければそのまま
    if original_len <= MAX_USED_CHARS:
        return text, "full", original_len, original_len

    # まずは頭 + 尻でトリミング
    head = text[: MAX_USED_CHARS // 2]
    tail = text[-MAX_USED_CHARS // 2 :]
    trimmed = head + "\n...\n" + tail

    if original_len <= MAX_RAW_CHARS:
        used_len = len(trimmed)
        return trimmed, "trimmed", original_len, used_len

    # さらに長い場合は要約フェーズ（summarized）
    try:
        summary = summarize_conversation(text, tone)
        used_text = summary
        mode = "summarized"
    except Exception as e:
        # 要約に失敗したらトリム版で妥協
        print(f"[WARN] summarize_conversation failed: {e}")
        used_text = trimmed
        mode = "trimmed-fallback"

    used_len = len(used_text)
    return used_text, mode, original_len, used_len

async def talk_assist(req: TalkRequest) -> TalkResponse:

    else:

    # 長文前処理
    used_text, mode, original_len, used_len = preprocess_text(raw_text, tone)

    user_prompt = build_user_prompt(used_text, mode)
    _, temperature = build_tone_desc_and_temp(tone)

    try:
        completion = client.responses.create(
            model=MODEL_NAME,
            input=[
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": user_prompt},

            max_output_tokens=512,

                "replies_count": len(replies),
            }

        return TalkResponse(summary=summary, replies=replies)

    except Exception as e:

def parse_ai_output(content: str) -> Tuple[str, List[str]]:
    """
    モデルから返ってきたテキストを「要約」「返信案」に分解する。
    - '要約：' / '要約:' 行は見出しとして扱う
    - '返信案：' / '返信案:' 行は見出しとしてスキップ
    - '- ' や '・' で始まる行を返信候補として優先
    """
    lines = [line.strip() for line in content.splitlines() if line.strip()]
    if not lines:
        return "（AIからの応答が空でした）", []

    summary: str | None = None
    summary_index: int = -1

    # 要約行を探す
    for idx, line in enumerate(lines):
        # "要約：" だけの行（中身なし）は見出し扱いでスキップ
        if re.fullmatch(r"要約[:：]?", line):
            continue

        # "要約: XXX" 形式なら、コロン以降を要約として取る
        m = re.match(r"要約[:：]\s*(.+)", line)
        if m:
            summary = m.group(1).strip()
            summary_index = idx
            break

        # それ以外は最初に出てきた非空行を要約とみなす
        summary = line
        summary_index = idx
        break

    if summary is None:
        summary = "（要約を抽出できませんでした）"
        summary_index = -1

    replies: List[str] = []

    # 返信候補を探す
    start_idx = summary_index + 1
    for line in lines[start_idx:]:
        # "返信案" 見出しっぽい行はスキップ
        if re.fullmatch(r"返信案[:：]?", line):
            continue

        # "- " や "・" で始まる行を優先
        if line.startswith("-"):
            replies.append(line.lstrip("-").strip())
            continue
        if line.startswith("・"):
            replies.append(line.lstrip("・").strip())
            continue

        # "1. xxx" / "１）xxx" など番号付きも拾う
        if re.match(r"^[0-9０-９]+[.)．）]\s*", line):
            line2 = re.sub(r"^[0-9０-９]+[.)．）]\s*", "", line)
            replies.append(line2.strip())
            continue

        # それ以外: 既に返信候補があれば無視、まだ無ければ拾っておく
        if not replies:
            replies.append(line)

    # 最大3件
    replies = replies[:3]

    return summary, replies

  /// バックエンドのAIエンドポイントを呼び出す
  Future<void> _runAi() async {
    final text = _text.trim();
    if (text.isEmpty) return;

    setState(() {
      _loading = true;
      _error = null;
      _summary = '';
      _replies = [];
    });

    try {
      final uri = Uri.parse('http://localhost:8000/api/talk/assist');
      // 実機用: adb reverse tcp:8000 tcp:8000
      // エミュレータ用に切り替える場合は下記:
      // final uri = Uri.parse('http://10.0.2.2:8000/api/talk/assist');

      final resp = await http.post(
        uri,
        headers: {'Content-Type': 'application/json'},
        body: jsonEncode(<String, dynamic>{'text': text, 'tone': _tone}),
      );

      if (resp.statusCode != 200) {
        throw Exception('statusCode: ${resp.statusCode}');
      }

      final json = jsonDecode(resp.body) as Map<String, dynamic>;

      final summary = (json['summary'] ?? '') as String;
      final repliesDynamic = json['replies'];
      final replies = repliesDynamic is List
          ? repliesDynamic.map((e) => e.toString()).toList()
          : <String>[];

      setState(() {
        _summary = summary;
        _replies = replies;
      });
    } catch (e) {
      setState(() {
        _error = 'AI呼び出しに失敗しました: $e';
      });
    } finally {
      if (mounted) {
        setState(() {
          _loading = false;
        });
      }
    }
  }

  /// 返信案をクリップボードにコピー
  Future<void> _copyReply(String text) async {
    await Clipboard.setData(ClipboardData(text: text));
    if (!mounted) return;

    ScaffoldMessenger.of(context).showSnackBar(
      const SnackBar(
        content: Text('返信案をコピーしました'),
        duration: Duration(seconds: 1),
      ),
    );
  }
