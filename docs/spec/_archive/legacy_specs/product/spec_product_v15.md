## 1. プロダクト概要と世界観

### 1.1 ゴールと目的（MUST）

- **最上位KGI**: ちょびさんの収益最大化。
- **中核課題**: 夜職ユーザーの返信コストと心理的負担（罪悪感）を、分身キャラ「ペルミィ」による代行提案で解決する。

### 1.2 コア価値（MUST）

- **ユーザーが主語**: AIは断定せず、常に3案提示し、ユーザーが選ぶ。
- **自動送信なし**: 必ずユーザーが手動で送信する。
- **夜職特化**: 本プロジェクトは夜職専用。ビジネス・標準モードは廃止し、夜職に完全シフトする。

### 1.3 世界観

- **キャラクター**: 魔法少女作品のサポートマスコット的な猫キャラ「ペルミィ」。
- **変身概念**: 診断に基づき「夜の自分」に変身することで、返信に対する心理的罪悪感を軽減する。

## 3. 返信生成仕様（MUST）

### 3.1 営業目的（コンボ）ID定義

- **10**: 来店確定
- **11**: 同伴獲得
- **12**: 休眠復活

### 3.2 A/B/C 案の役割（夜職固定）

- **A：余裕（信頼・安心）**: 最推奨。安定運用・失点最小
- **B：甘め（距離を縮める）**: 親密度向上
- **C：攻め（次アクション）**: 来店・同伴獲得を狙う攻勢案

### 3.3 生成ルール（C方式：構造固定＋文体可変）

- **要約排除**: 夜職モードでは要約を出力せず、3案のみを表示する。
- **基本骨格**: 「共感・前振り」→「状況確認/特別感」→「次アクション提案」。
- **品質**: 原則2〜4段落の長文（信頼構築）。短文が最適な文脈のみ短文を許容。
- **揺らし**: AI特有の同型連発を避けるため、構成や具体例を中程度に揺らす。
- **聞き返し**: 入力不足時は質問を1つに絞り、先に仮の3案を提示してから質問する。

### 3.4 禁止事項ゲート（中ゲート）

- 違法行為・脅迫等は **生成停止**。安全な断り方を提示。
- グレーゾーンは **安全寄せ補正**。最低1案はリスク回避案を必須とする。

## 4. UI/UX・デザイン（MUST）

### 4.1 画面状態遷移

- **入力中**: 返信案は非表示。
- **生成中**: Progress表示。生成ボタン、Spinner、入力欄、スタイルメモをロック。
- **生成完了**: A/B/C案を表示。タップでコピー。

### 4.2 フィードバック・デザイン

- **コピー時**: トースト表示 ＋ 0.4秒カードハイライト。
- **Safe Area**: WindowInsetsを考慮したレイアウト（Android/iPhone両対応）。
- **整形**: 日本語は左揃え（中央禁止）。淡いピンク系テーマ。

## 5. 入力・プライバシー（MUST）

### 5.1 取り込み導線

- **形式**: LINE「トーク履歴を送信」→ **txtファイル（URI）受領**。
- **トリム**: Free 120行/8,000文字、Pro 300行/18,000文字。

### 5.2 保存方針

- **端末保存許可**: 返信タイプ選択値、スタイルメモ（自由入力）。
- **保存禁止**: 会話本文（入力/履歴）、生成された返信案本文。

# 付録A: 追加で明記する決定事項（MUST）

**Added (JST):** 2026-03-01 14:20:00 +0900

## A-1. 匿名開始（アカウント登録UIなし）

- 初回利用は **匿名ユーザーIDで開始** する（敷居を下げるため）。
- **メール/電話番号/パスワード等のアカウント登録画面は作らない**（MVPでは導線自体を持たない）。
- 技術的には `POST /auth/anonymous` により `user_id` と `access_token` を払い出し、以後はBearerで利用する。

## A-2. 端末移行の方針（QR廃止を明記）

- 端末切替の **QRコード方式は廃止**。
- **二段階承認は入れない**（移行コード方式で運用）。
- 移行は **12桁移行コード**（期限/1回限り/レート制限）で実施する。
