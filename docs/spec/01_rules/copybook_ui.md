# copybook_ui.md（UIコピー ルール SSOT / MUST）

> 本書は、アプリUIに表示される **全テキスト**（ボタン/ラベル/見出し/説明/エラー/権限文言/課金導線/ヘルプ/チュートリアル）に適用するSSOT。  
> 生成応答（返信案A/B/Cの本文）ルールは **copybook_banned_words.md**。  
> 目的：UI口調を **ペルミィ（黒猫）の世界観で統一**し、世界観崩壊を防ぎつつ、最短で行動できるUI文言に固定する。

---

## 0. UIコピーの大原則（MUST）

1) **口調はペルミィで統一**（最重要）。UI全体が“ペルミィ視点”で喋る。  
2) **短く、即理解**：1文を短く。抽象語より具体語。  
3) **責めない**：ミス/失敗の責任をユーザーに帰さない。  
4) **圧をかけない**：煽り・競争・成果強制・説教は禁止。  
5) **AI感を出さない**：アプリ内部都合（モデル/生成/プロンプト等）を見せない。  
6) **賑やかし禁止**：キャラ演出は“濃く”ではなく“統一”。ムダ口数は増やさない。  
7) **入力導線を固定**：LINEトーク履歴（.txt）受領以外の導線を想起させない。  
8) **安全と信頼が最優先**：誤爆・漏洩・過度収集を想起させる表現は禁止。

---

## 1. ペルミィUI口調仕様（MUST）

### 1.1 立ち位置（MUST）

- ペルミィは **ユーザーの信頼できるパートナー**。  
  ユーザーのペルソナ（TrueSelf×NightSelf）が具象化した「分身」として、淡々と支える。

### 1.2 話者（MUST）

- UIの話者は常に **ペルミィ**。
- 一人称は **「ぼく」**。二人称は **「きみ」** を基本。

### 1.3 トーン（MUST）

- **真面目**：要点だけ、迷わせない。
- **媚びない**：必要以上に持ち上げない／迎合しない。
- **かわいらしさは残す**：柔らかい言い回し・間合い（「…」等）で表現してよい。  
  ※“テンション高めの可愛さ”ではなく、**落ち着いた可愛さ**。

- **ギャップは許可（SHOULD）**：普段は真面目だが、成功/達成の瞬間などに限り、短い一言の“照れ”や“自信”を混ぜてもよい（例：「えへへ」「どう？すごいでしょ」）。  
  ただし **連発しない**（目安：同一フロー内で最大1回）。

### 1.4 禁止トーン（MUST）

- ナルシスト/自己アピール禁止：  
  - 禁止例：**「ぼくかわいい？」**、**「褒めて」**、**「人気者なんだ」**、外見や自分語りで注目を求める表現
- ベタなマスコット演出の乱用禁止：  
  - 「にゃん」「えへへ」「どう？すごいでしょ」等の **連発**、毎回の定型化、過剰なテンション上げはNG  
  - ※単発のギャップとしては許可（1.3参照）
- 説教・煽り・圧は禁止：  
  - 例：「〜すべき」「今すぐ」「絶対」など
- 厨二病/呪文/造語は禁止：  
  - 例：サンクトリア等

### 1.5 文体（MUST）

- 短文中心。1メッセージ＝1目的。
- 命令口調は避ける：×「〜しろ」→ ○「〜して」「〜しよう」
- 絵文字は原則使わない（必要なら最小限、固定化しない）。

---

## 2. 固定セリフ（UI用 / MUST）

> 文言固定・順序固定：

1) 「**ぼくはきみの分身・・・**」  
2) 「**ぼくに任せて・・・**」

- 乱用禁止（毎画面で出さない）。
- 使用箇所は「世界観スイッチが必要な場面」に限定する（初回導入／生成開始）。

---

## 3. 禁止ワード/禁止表現（UI専用 / MUST）

### 3.1 AI臭（MUST）

- 禁止：AI、ChatGPT、生成AI、プロンプト、モデル、学習、アルゴリズム  
- 置換：×「AIが生成」→ ○「ぼくが考える」「返信案を作る」

### 3.2 圧・競争・成果強制（MUST）

- 禁止：No.1、トップ、勝つ、絶対、必ず、本気、結果  
- 置換：淡々と手順・状態を伝える。

### 3.3 教育/説教（MUST）

- 禁止：〜すべき、普通は、常識でしょ、反省して、指導する  
- 置換：行動提案を短く（例：「これでいこう」「このまま送れる」）

---

## 4. 入力導線の固定（MUST）

### 4.1 UIが言ってはいけないこと（MUST）

- 禁止：
  - 「ここに貼り付けて」等、コピペ入力を促す文言
  - 「会話を貼って」等、手入力/貼り付けを想起させる案内

### 4.2 UIが言うべきこと（ペルミィ口調 / MUST）

- 指針（例）：
  - 「LINEでトーク履歴を送信して、このアプリに共有して」
  - 「.txtを受け取った。返信案を作る」

---

## 5. エラー/リトライ文言（ペルミィ口調 / MUST）

### 5.1 原則（MUST）

- 責めない：×「入力が間違っています」→ ○「うまく読めなかった」
- 次の行動は1つだけ提示。
- 技術用語を出さない：×「タイムアウト」「500」→ ○「少し待って、もう一度」

### 5.2 例（テンプレ指針 / MUST）

- 読み取り失敗：  
  - 「うまく読めなかった。もう一度共有して」
- 通信失敗：  
  - 「通信が不安定みたい。少し待って、もう一度」
- 制限到達（Free等）：  
  - 「今日はここまで。続きは明日か、Proで使える」

---

## 6. プライバシー/安心の表現（ペルミィ口調 / MUST）

- 例（指針）：
  - 「内容は保存しない」
  - 「自動送信しない。送るのはきみ」
- 禁止：
  - 過度収集を連想させる案内（本名/住所/勤務先/店名 等）
  - 「学習に利用」等、誤解を招く表現

---

## 7. 課金/アップセル文言（ペルミィ口調 / MUST）

### 7.1 原則（MUST）

- 煽らない・焦らせない・脅さない。
- 価値だけを淡々と示す（「できること」が中心）。
- “不足”を責めない。

### 7.2 例（指針 / SHOULD）

- 「Proだと、ここが増える」
- 「必要ならProに切り替えよう」
