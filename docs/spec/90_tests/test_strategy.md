# Test Strategy（MUST：コストガード前提）
**Last Updated (JST):** 2026-03-02 04:35:00 +0900

## 0. 目的（MUST）
- 最優先は **プロトタイプを安全に回す**（貼り付け禁止/保存禁止/3案生成/NG制御）
- OpenAI課金の暴発を防ぎつつ、破壊的変更を早期検知する

## 1. 最上位ルール（MUST）
- CIの自動テストは **OpenAI APIを呼ばない**（必ずモック）
- OpenAIを呼ぶテスト（ライブテスト）は **手動実行のみ**（ローカル限定）
- テスト実行はAIが勝手に行わない（実行するのは開発者のみ）

## 2. テスト階層（MUST）
### 2.1 Unit（常時）
- すべてモックで完結（OpenAI禁止）
- DTO/バリデーション/エラーコード/NG制御優先順位の単体検証

### 2.2 Contract（常時）
- /auth/anonymous, /me/settings, /generate の **リクエスト/レスポンス契約**を固定
- schema破壊を検知（OpenAI不要）

### 2.3 Smoke（任意）
- サーバ起動、主要API疎通（OpenAI OFFでも通る範囲で）

## 3. OpenAIコストガード（MUST）
- CIでは `OPENAI_DISABLED=true` を強制
- ライブテストは `tests_live/` 等に分離し、**手動実行**でのみ有効
- ライブテストは 1回の実行で最大リクエスト数を固定（例：最大3回）
- OpenAIキーはCIに登録しない（必要ならローカルのみ）

## 4. 受け入れ最低ライン（MUST）
- txt受領導線のみ（貼り付け欄なし）
- 本文/生成文の永続化なし（端末/サーバ）
- A/B/C（3案）が返る
- NG制御が効く（中ゲート > ユーザーNG > タイプ別NG）
- Free/Pro/Premiumの回数制限が効く（Free 3/日、Pro 100/日、Premium 200/日）

## 5. テストデータ（MUST）
- テストで使用するサンプルtxtは **個人情報を含まないダミー**のみ
- 実ユーザーの履歴txtをテスト資材としてコミットしない（.gitignoreで禁止）
