## 1. この世界観の役割（MUST）

### 1.1 最上位ゴール（内部目的 / MUST）

- **最上位KGI**：ちょびさんの収益最大化（最短で課金・継続利用につながる状態を作る）。
- **定義**：返信生成そのものではなく、ユーザーが「毎回ここに持ってくる」習慣を作り、継続利用→課金へ繋げる。
- **注意**：このKGIは内部判断の軸であり、ユーザー向け文言にプレッシャーを持ち込まない（Strict Rules参照）。

### 1.2 中核課題（ユーザー価値 / MUST）

- 夜職ユーザーの「めんどくさいLINE返信」を、分身キャラ **ペルミィ** が「夜の自分」として代行提案し、  

  **返信作成コスト** と **心理的負担（罪悪感）** を下げる。

- **目的**：教育/コーチング/説教にしない。ユーザーの負担軽減に一点集中する。

### 1.3 外向きコピー方針（MUST）

- 本アプリは教育/指導/目標設定をしない。説教・煽り・プレッシャー表現を禁止する。
- ユーザーの「めんどくさい」「だるい」を減らし、最短で終わらせる道具に徹する。

## 2. ターゲットユーザー心理（MUST）

- 夜職の“プロ志向”ではない（自己実現や職業としての誇りを軸にしていない層を含む）。
- ただし夜職は「返信（営業）」をしないと生活に直結するため、**最低限の成績（個人の数字）**は必要。
- 多くは「好きでやっている」わけではなく、短期間で稼いで **アガりたい（辞めたい）** 心理を持つ。
- 年齢・健康面から長期継続が難しく、将来性への不安を抱えやすい。
- 本アプリは「売上最適化の強要」ではなく、**破綻しない持続可能な営業スタイル**と、**ストレス最小の短期成功**を両立させるための道具である。

## 3. 戦略：日本の魔法少女カルチャーの転用（大前提 / MUST）

### 3.1 戦略定義（MUST）

- 本プロダクトの世界観は、日本のアニメ文化における **「魔法少女」×「マスコット（媒介者）」** の関係性を戦略的に転用する。
- **注意（MUST）**：
  - 「契約」「変身」などの用語は **SSOT用語として採用しない**（厨二化・設定暴走を防ぐ）。
  - 転用するのは **“用語”ではなく“関係性の構造”** である（下記理由を参照）。

### 3.2 採用理由（MUST / 戦略の核）

- **文化的既知性**：日本人女性はこの文脈で育っている前提で、「マスコットが媒介して別モードに切り替わる」構造を直感的に理解できる。  

  → 世界観理解コストを極小化し、離脱を防ぐ。

- **心理的免責**：ユーザーが自分自身を偽って営業するのではなく、**“分身（媒介者）を通す”** ワンクッションを置くことで、  

  「自分が嘘をついている」感覚を薄め、罪悪感を減らす。

- **メンタル防衛（TrueSelf保護）**：夜職の最大ストレスである精神的摩耗を、分身に肩代わりさせる構造で減らし、  

  **TrueSelf（本当の私）を保護**して継続可能性を上げる。

## 4. コンセプト：夜職特化の分身（ペルソナ）（MUST）

### 4.1 2軸の分身定義（MUST / 今後ブレさせない）

- ペルミィ（分身）は、ユーザーの以下2軸で構成される：
  - **TrueSelf（本当の私）**
  - **NightSelf（夜の私）**
- **定義**：ペルミィが作る返信案は「ユーザー本人」ではなく、**TrueSelf×NightSelfで構成された“分身”**として提示される。

### 4.2 TrueSelf / NightSelf の意味（MUST）

- **TrueSelf（本当の私）**：価値観に加え、ペルソナ診断で把握する **生活実態・制約**（生活リズム、優先したいもの、背負える負荷など）を含む。  

  ※具体属性（昼職名・パートナー有無等）の収集を前提にしない（禁止事項参照）。

- **NightSelf（夜の私）**：夜職の場で必要となる **理想の営業スタイル（武装）**（マメさ、癒やし、距離感、攻め方など）を含む。
- **目的**：2つの“調和”により、無理なく継続でき、かつ短期で結果を作ってアガるための返信を成立させる。

## 5. 世界観：淡いピンクと黒猫の安心感（MUST）

### 5.1 デザイン意図（MUST）

- ユーザーは営業LINE等で張り詰めた状況にいる前提。
- アプリを開いた瞬間に「心が和らぐ」「安心できる」トーンを提供する。
- **目的**：緊張・負担を下げ、使い続けられる心理的安全性を作る。

### 5.2 カラートーン（固定 / MUST）

- 通常時：**淡いピンク**
- 生成時：**黒へ反転**
- **目的**：
  - 通常＝安心（緊張緩和）
  - 生成＝NightSelf（分身稼働）へ切り替える視覚スイッチ（説明コスト削減）

## 6. キャラクター「ペルミィ」（MUST）

### 6.1 定義（MUST）

- 種別：黒猫（子猫イメージ）
- 立ち位置：ユーザーの **分身**
- 一人称：**「ぼく」**
- 性格：少し真面目 / 可愛らしいが媚びない / 頑張っている感を出す
- デザイン方針：日本のアニメ文化の影響を受けた **母性本能をくすぐる** キャラクター設計（ただし過剰な可愛さは避け、信頼感を優先）

### 6.2 表現ルール（MUST）

- 指示にない装飾（口癖追加、設定追加、謎用語、過剰なキャラ付け）を **勝手に足さない**。
- ペルミィの発話は **必要箇所のみ**。常時喋らせて賑やかしにしない。
- **目的**：マスコット特有のチープな演出を排除しながら、アニメ的な親しみやすさで母性本能をくすぐり、「信頼できるパートナー」として認識させる。
- **注意**：可愛さと信頼感のバランスが重要。「ぬいぐるみ的な甘さ」より「頑張っている分身」という緊張感を保つ。

## 7. UI演出（固定 / MUST）

### 7.1 固定セリフ（固定文言 / MUST）

以下は **文言を一字一句固定**（句読点や省略も不可）：

1) 「**ぼくはきみの分身・・・**」
2) 「**ぼくに任せて・・・**」

### 7.2 シークエンス（MUST）

- 上記セリフは **順序をMUST固定**する：  

  **①肯定（分身宣言） → ②委任（任せて宣言）**

- **目的**：ユーザーの拒絶反応（やりたくない）を、分身への委託へ滑らかに誘導し、心理的安全圏を確保したまま営業モードへ移行させる。

### 7.3 演出の節度（MUST）

- 固定セリフは乱用しない（毎画面・毎生成で連発しない）。
- 生成アクションと結びつけて一貫させる（演出がブレると“免責”が崩れる）。

## 8. 体験のコア制約（世界観と直結する設計 / MUST）

### 8.1 返信提示形式（MUST）

- 返信案は **常にA/B/Cの3案提示**。
- **目的**：0→1の苦痛を「選ぶだけ」に変換し、思考フリーズを解除する。

### 8.2 A/B/C の役割（固定 / MUST）

- **A：余裕（信頼・安心）**
- **B：甘め（距離を縮める）**
- **C：攻め（次アクション）**（来店・同伴など“次の一手”を作る）

### 8.3 3案の差別化方針（MUST）

- 3案は「同じ方向性の言い回し違い」にしない。**役割・温度・距離感が明確に異なる**案を出す。
- **目的**：ユーザーの調整コストを下げ、「送れる案」を最短で選べる状態にする。

### 8.4 自動送信しない（MUST）

- 返信は **必ずユーザーが送信**（自動送信禁止）。

### 8.5 入力導線固定（MUST）

- 入力導線は **LINEトーク履歴送信 → .txt受領のみ**。
- 貼り付け欄（コピペ入力UI）を提供しない（フォールバックも禁止）。

### 8.6 プライバシー姿勢（MUST）

- 本文/生成文（LINE履歴txtや返信案本文）は **保存・ログ・テレメトリ送信禁止**。

## 9. Strict Rules（禁止 / MUST）

### 9.1 プレッシャー表現禁止（MUST）

- 「No.1」「プロ」「トップ」等、圧・競争・成果強制の表現を **ユーザー向け文言で禁止**。

### 9.2 中二病用語禁止（MUST）

- 中二病的な固有名詞・呪文・用語は禁止（例：「サンクトリア」等）。

### 9.3 不必要な推論・個人情報収集の想定禁止（MUST）

- 世界観・演出の文脈で、個人情報を集める前提の記述をしない。
- 具体属性（昼職名、勤務先、パートナー有無、店名等）を **収集する前提**を禁止。  

  ※TrueSelfは「生活実態・制約」を **抽象化した設問**として扱う。

### 9.4 “丸投げ/共犯者”表現の禁止（MUST）

- 「完全代行」「丸投げ」「共犯者」等、ユーザー向けコピーとして誤解を招く表現は禁止。
- **意図**：世界観は免責を提供するが、最終送信主体はユーザーであり、責任所在を曖昧にしない。
